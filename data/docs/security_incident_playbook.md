# Security Incident Playbook

Employees must report suspected security incidents within 15 minutes in the `#security-help` Slack channel and by filing a Sev intake form. Examples include phishing attempts, lost devices, credential theft, or unusual system activity.

Managers are responsible for making sure the affected employee stops using the compromised system, preserves evidence, and escalates to the on-call security lead. Managers should not ask the employee to continue working from the affected device until security approves it.

The security lead determines severity within 30 minutes, documents actions in the incident tracker, and decides whether to notify legal, IT, or executives.

Post-incident reviews are required for all Sev-1 and Sev-2 events within five business days. The review must include timeline, root cause, customer impact, and prevention actions.
